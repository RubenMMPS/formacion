1. ¿Qué es un linter?
Es una herramienta que analiza tu código para detectar errores potenciales, malas prácticas o falta de adherencia a un estilo de escritura (como punto y coma faltantes o variables no usadas).

2. ¿Qué es Husky?
Es una herramienta de desarrollo que automatiza tareas, como linting o pruebas unitarias, ejecutándolas automáticamente antes de confirmar cambios o subir código a Git.

3. ¿Qué es la reactividad?
Es la capacidad para responder de forma automática a cambios en los datos de una web.

4. ¿Desde dónde hay que crear todas las ramas para los desarrollos?
Normalmente, todas las ramas para los desarrollos deben nacer desde la rama main.

5. ¿Cuándo se puede utilizar la rama main?
La rama main solo se utiliza para:

Recibir merges de ramas probadas y aprobadas por una o más personas.

Hacer hotfixes (correcciones urgentes de errores en producción).

Etiquetar versiones (releases).

6. ¿Qué son las custom properties?
Son entidades que contienen valores específicos para ser reutilizados en todo el documento.

7. Diferencias entre v-if y v-show
La principal diferencia es que si la condición no se cumple para ejecutar su interior en el v-if deja de renderizarse directamente, mientras que en el v-show se sigue renderizando y únicamente le pone el atributo de display:none.

8. ¿Cuáles son los valores de falsy?
Los valores de falsy son: "false", "0", "-0", ""(string vacio), null, undefined, NaN(Not a Number) y 0n ( big integer cero).

9. ¿Qué es Nullish Coalescing Operator?
Es un operador lógico que devuelve el operando de la derecha cuando el de la izquierda está vacio, lo que quiere decir null o undefined.

10. ¿Para que se utiliza TypeScript?
Se utiliza para programar en un lenguaje como JavaScript pero tipado, lo que significa que te permite visualizar errores antes de ejecutar el código y hacer que el mantenimiento de aplicaciones sea mucho más seguro y predecible sin que se te cuele ningun tipo de valor que no quieras.
